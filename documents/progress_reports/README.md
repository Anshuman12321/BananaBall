# Progress reports

This folder is for **weekly progress artifacts** that we can quickly pull up during meetings (demos, diagrams, screenshots, API summaries, etc.).

## Folder structure (weekly)

Create a new folder each week using a clear date prefix:

- `MM_DD Report Files/`
  - `Backend/`
  - `Frontend/`
  - `Notes/` (optional)

## What to put in each week

- **Backend**
  - API endpoint list/export (HTML/PDF)
  - diagrams (data model, flowchart)
  - screenshots of Swagger (`/docs`) if helpful
  - short notes about what changed / what’s next
- **Frontend**
  - screenshots / screen recordings
  - component/UI notes
  - any relevant build or dev-server notes
- **Notes (optional)**
  - meeting agenda, action items, risks/blockers

## Naming tips

- Prefer short, descriptive filenames: `Backend_API_List.html`, `Draft_Flow.png`, `Week_2_Notes.md`, etc.
- If an artifact is a snapshot of the running app, include the date in the name.
